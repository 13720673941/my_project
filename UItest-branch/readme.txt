【超级售后-滴答服务网点PC端自动化测试脚本】

------------------------------------------------------------------------------------------------------------------------
python + selenium + unittest + ddt 框架搭建 ---基于POM的设计模式,结合Json数据文件格式 Page Object Model
------------------------------------------------------------------------------------------------------------------------

文件夹：

[Common]:     封装公共的函数、类文件夹,包括：1、createUser.py封装随机生成注册使用的用户名和手机号,2、driver.py封装浏览器驱动
              3、getData.py封装获取文件路径函数DataDirPath,获取json数据文件函数GetJsonData,4、logConfig.py logging模块的二次封装
              5、rwConfig.py封装ini配置文件的读取和写入,6、screeShot.py截图文件,7、writeTestResult.py封装写入测试结果到ini配置文件

[Data]:       测试数据存放文件夹,包括：1、TestData.json所有测试用例的数据存放文件json格式,2、NewRegister.txt自动化注册的网点用户名、账号、密码信息
              3、RegPhoneNum.txt随机生成的手机号保存到该文件,避免重复生成,4、RegUserName.txt随机生成的用户名保存到文件夹,避免重复

[Page]:       每个页面的操作封装

[Result]:     测试结果,包括：1、ErrorPicture错误截图的保存文件夹,2、Log日志文件的保存文件夹,3、Report测试报告的存放文件夹,
              4、TestResult.ini测试结果的保存文件

[Run]:        运行测试脚本文件夹

[Script]:     测试用例脚本保存文件夹

------------------------------------------------------------------------------------------------------------------------

脚本执行顺序：Run运行脚本->Script脚本内容->调用Common下的getData.py读取测试数据json文件->调用Page文件夹对应的页面信息操作->
             ->写入测试结果到TestResult.ini->生成测试报告文件到Report,日志文件到Log文件夹下
