【超级售后网点接口测试脚本】
------------------------------------------------------------------------------------------------------------------------
python + requests + unittest + excel 框架搭建
------------------------------------------------------------------------------------------------------------------------
脚本执行：读取Excel->判断读取config配置文件->判断Url信息->判断请求头信息->判断读取json请求数据文件->对请求数据中的固定参数进行赋值->
        调用requests模块发送请求->判断接口测试结果->写入日志文件->写入测试结果到配置文件->输出测试报告
------------------------------------------------------------------------------------------------------------------------
文件夹：

[Common]:   公共函数、类文件夹提供后面测试脚本的调用,包括->日志的封装、excel文件的读写、config配置文件的读写、json文件的读写、
                                              requests模块的二次封装、测试结果的写入函数、Public个函数之间的调用函数、
                                              Variable对Excel文件中的用例的列数进行变量赋值提供后面excel的读取、写入调用
                                              Page对象层,对动态请求参数的特殊处理关联配置文件公用数据和json之间

[Data]:     公共的api使用文件,包括->BaseConfig基本配置文件接口公用的数据、BranchApi接口测试用例集Excel表格、RequestData接口测试的请求
            数据json格式

[Log]:      存放每个脚本的执行日志,TestResult每个脚本的执行结果写入配置文件

[Report]:   脚本执行结果的测试报告输出文件夹

[Run]:      执行用例集合

[Script]:   脚本存放文件夹
------------------------------------------------------------------------------------------------------------------------
用例：

test_login.py       超级售后网点登录接口的测试脚本
test_overview.py    网点首页概览模块的所有接口请求
test_orderflow.py   网点自营订单派单、预约、改约、反馈、取消订单接口
test_searchorder.py 网点订单查询模块各种订单状态订单查询
test_questorder.py  网点异常订单模块,待处理、已处理、处理中的异常订单查询
test_JDorder.py     网点鉴定单模块提交鉴定单、已鉴定、待鉴定订单查询
test_THorder.py     网点整机退换模块,提交整机退换订单、各种整机退换订单状态订单信息查询

------------------------------------------------------------------------------------------------------------------------

Excel数据文件的格式：

Id:             用例的编号信息,模块+编号
Case:           对接口用例的简单描述信息
Url:            接口的请求地址信息,公共的部分+后续url数据,有公共部分就写Url1/Url2/Url1参考requests模块中对url读取后的判断来写
Method:         请求的类型,一般为POST/GET
Headers:        请求头信息,需要token的就填写Authorization,需要apitoken的填写接口请求的标签sign以字典格式,都不需要的就不填写Headers信息
Body:           请求数据,有请求数据的填写用例编号信息,没有的不填写
Code:           协议状态码一般为200
ResponseCode:   业务状态码
Exception:      期望的请求返回的数据中包含的参数信息
Result:         测试结果信息,一般为Pass/Fail自动写入
ContentType:    请求数据类型的表单=form,json=json,固定格式,后面有对数据类型的判断,牵扯到请求参数的序列化和反序列化的情况

------------------------------------------------------------------------------------------------------------------------

脚本依赖关系：

脚本                    依赖脚本                                                              程度
------------------------------------------------------------------------------------------------------------------------
OperationExcel.py      Variable中对excel中列数的变量赋值处理                                    每次调用
------------------------------------------------------------------------------------------------------------------------
OperationJson.py       OperationExcel读取的Excel中用例的body值读取api的body数据                 每次调用
------------------------------------------------------------------------------------------------------------------------
Requests.py            Excel、Json、Config读取的请求数据,特殊需要请求参数动态的话需要依赖Page    每次调用,page个别动态关联的请求需要
------------------------------------------------------------------------------------------------------------------------
test_overvie.py        test_login执行写入配置文件的token等数据                                  每次调用
